## 为什么是 agent-terminal？

测试 CLI 工具很难。你可以检查退出码，捕获 stdout/stderr，但你看不到终端里到底发生了什么。提示符渲染正确吗？加载动画转起来了吗？用户的输入正常回显了吗？

**agent-terminal 基于一个简单的理念：终端这个黑箱，本不该是黑的。**

通过透明代理 zsh PTY（伪终端），我们捕获每一次交互的每一个字节——每一次按键、每一个转义序列、每一次屏幕刷新——让你能从外部观察、注入命令、并对完整的终端状态进行断言。

## 核心理念

### 可观察性优先

测试不应该要求修改被测代码。agent-terminal 坐在 shell 和操作系统之间，捕获一切而目标应用浑然不觉。没有 mock，没有 hook，只有透明的观察。

### 可编程交互

手动测试无法规模化。有了 agent-terminal，你可以用代码描述交互：启动会话、等待提示符、注入命令、断言屏幕内容。终端变成了一个可编程的接口。

### 快照无形之物

终端状态是转瞬即逝的——直到你捕获它。`dump` 命令给你一个时间点的精确屏幕视图，正是用户会看到的模样，包括光标位置、颜色和格式。无需猜测，调试渲染问题。

## 你能做什么

**会话管理**
- 启动持久的、可观测的 zsh 会话
- 列出并管理多个并发会话
- 会话终止时自动清理

**交互控制**
- 向运行中的会话注入按键和命令
- 捕获原始 PTY 输出和解析后的屏幕状态
- 通过心跳机制查询会话健康状态

**测试与自动化**
- 验证提示符渲染和 UI 行为
- 自动化交互式 CLI 工作流
- 对终端屏幕内容进行断言

## 为谁而建

agent-terminal 是为这样的开发者准备的：
- 构建交互式 CLI 工具（TUI 应用、提示符、加载动画）
- 需要测试终端渲染行为
- 希望自动化端到端 shell 工作流
- 厌倦了终端应用的"在我机器上能跑"

## 愿景

我们相信终端应用应该享有与 Web 或移动应用同等的测试严谨性。agent-terminal 的未来包括声明式测试脚本、实时输出流、以及连接远程会话的分布式测试运行器——将终端从测试盲区转变为完全可观测、可测试的表面。
